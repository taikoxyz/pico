Overall Verdict: NOT READY for mainnet production launch

The protocol design and contract code are sound (v2 brings on-chain HTLC settlement to parity with Lightning's trust model), and the codebase is well-structured with meaningful test coverage. However, deployment infrastructure is incomplete, all human operator gates are PENDING, and no real-USDC mainnet smoke test has been completed. The project needs approximately 4–6 weeks of focused operational work before GA.

2.2 Missing Tests (by severity)

CRITICAL C-01: No test for watchtower offline → stale-state steal

The threat model's primary attack vector (user with no watchtower for 24h) has no automated test. Add e2e: open channel → sign v2 → user submits v1 (stale) → kill watchtower → 24h passes → verify stale state is finalized.

CRITICAL C-02: No ETH penalty proof test

PaymentChannel.eth.t.sol exists but doesn't test penalty path. ETH call{value:} disbursement has different semantics from ERC-20 transfer. Add PaymentChannel.eth.penalty.t.sol.

HIGH H-01: No cross-channel Update replay test

The Update wrapper's channelId prevents cross-channel replay, but no test verifies this. Add to state-machine/replay.test.ts.

HIGH H-02: No regression test for hub indexer gap (round-2 finding)

Hub missed on-chain events during restart in round-2 smoke. Add test: start hub → open channel → kill hub → open channel via contract directly → restart hub → verify hub discovers missed channel.

HIGH H-03: No CLI/SDK ETH payment E2E test

Round-2 smoke found ETH lifecycle "blocked by CLI/SDK gap." Add e2e/src/scenarios.eth.test.ts: open → topUp → pay → close with ETH.

MEDIUM No test for closeUnilateralFromOpen (anti-hostage), dust-griefing validation, or concurrent HTLC settlement on same channel.

LOW No test for CLI commands (the v1 agent surface has zero test files), operator token rotation, preimage cache TTL, or metrics format validation.

4. Competitive Analysis Refresh

The existing docs/competitive-analysis.html compares pico against 6 systems. This audit recommends adding 5 new systems and updating stale data.

4.1 Status Updates for Existing Systems

  • Lightning: Taproot Assets mainnet (Oct 2025) enables stablecoin issuance. ~13K nodes, ~5,300 BTC. update: no longer BTC-only.
  • Raiden: Effectively dead. Last release v0.4.0 (Sep 2024). No adoption.
  • Connext/Everclear: Pivoted to intent-based bridging in 2023. No longer state channels.
  • XRP Ledger: SEC resolved ($50M, Aug 2025). RLUSD stablecoin launched (Dec 2025). Payment channels unused.

Key competitive insight

x402 is the most direct competitor — same use case (AI agent payments), higher-adoption chain (Base). Its 1% fee is 50× higher than pico's 0.02%, giving pico a massive cost advantage at scale — but only if users accept channel setup friction. No competitor matches pico's fee structure for USDC micropayments.

  • PTLCs: Replace SHA-256 HTLCs with secp256k1 point commitments. Fixes passive correlation oracle from HtlcClaimed events.
  • Stealth addresses: Break chain-graph analysis. Scaffold exists, needs wiring.
  • Fee bucketing: Collapse adjacent payment amounts on-chain. Scaffold exists.
  • nostr ephemeral keys: Prevent relay-level linkability. Scaffold exists.

5.4 Business & Adoption Strategy

Target use cases: AI agent-to-agent payments → DVM/client payments → API metering → micro-tipping.

Key integrations: LangChain, CrewAI, ElizaOS, Nostr DVMs, Taiko ecosystem.

Positioning: "The Stripe for AI agents on Taiko." Compete on fee structure (0.02% vs. 1% for x402, 2.9% for Stripe). Accept UX friction of channel setup in exchange for massive fee advantage at scale.
